Building the Site
From development to deployment.

Summary

This site utilizes AWS S3's static webhosting capabilities. The setup was simple, well documented, and the maintenance is dirt cheap.

I wanted to create a personal site back in senior year. I didn't realize the scope of AWS back then, so I had a pretty naive approach of spinning up EC2 instances. I played around a bit, but ended up spending around $20 just for keeping my instance running. Enter my discovery of static web hosting. This service costs 50 cents a month! I placed my html/css from my original Django project, registered a domain and some necessary routing records in Route 53, and voila! The tutorial on setting this up had me manually logging into the S3 console, and dragging and clicking all the files I wanted to host. This is pretty inconvenient for when I shift a font awesome logo 2 pixels to the left for the fourth consecutive time, so I created a utility script to upload my files via a boto3 client in Python. A very important future improvement I need to make is to push a CDN in front of this and add some proper ACLs to my bucket. Why do I need this? Because literally anyone can push any changes they want if they can deduce the index file (and they'll probably figure out pretty quickly that it isn't "error.html") or any subpaths. Another (large) thing I need to do is properly rework my css to integrate flex boxes, as it's currently terrible on mobile.

I originally designed this site around a "personal project a month" theme focused on exploring different facets of development. The buttons in the home page each was supposed to be months, with the topics being whatever I found interesting during that time. I centered my naming conventions accordingly, calling my homepage the "Surface" and entries diving into topics "caverns".

I mean I'm still going to do that, but with way less ridiculous timelines. I'll probably overhaul the design too since the purpose of my site has changed. This place is in dire need of some cards.
